Add to an Existing Project

To add UX Aspects to your Angular CLI or Nx workspace, run the following command in the project root: 

ng add @ux-aspects/ux-aspects-ng1 --registry https://svsartifactory.swinfra.net/artifactory/api/npm/company-central-npm/
Note: If the ng add command reports a 404 error, ensure that your project uses version 9.0.4 or greater of @angular/cli.

Manual Installation

UX Aspects (AngularJS) components can be used in AngularJS projects, and those shown in the CSS section can be used in non-Angular projects. Details on installing and using the components are shown below.

  1. Install Node.js, which provides the NPM package manager. Select the LTS version when downloading.
  2. Install the UX Aspects (AngularJS) package and its dependencies: npm install @ux-aspects/ux-aspects-ng1 --save

The AngularJS components can also be used in a hybrid Angular app. However, please check the UX Aspects documentation for a native Angular version of the required component before taking this option.

  • If the component is marked Angular Compatible in the documentation, you can use it by importing HybridModule into the module where it will be used.

    The component name remains the same; however, the following should be considered when using this component in an Angular template:

    • Property names use camel case, so a property named on-next becomes onNext.
    • Angular binding syntax must be used, so a property with variable or function binding named valid would be written as [valid] for one-way binding, or [(valid)] for two-way binding (if supported).

  • Otherwise, the component should be upgraded using Angular's UpgradeAdapter.
